資料上傳：依流程分開

原本系統是分開步驟：先在「租金計算」分別產生上期、下期租金試算；月結總帳再吃這兩份試算結果。這裡已改成分區上傳；先做哪個流程，就只準備那一區需要的檔案。

① 租金計算用檔案

這一區對應原本租金計算頁。必要資料是：租金表、出入庫記錄表、重量資料。重量原本可從設定頁 / Supabase 載入，這個獨立版先用 Excel 上傳。

租金表 (.xlsx)期初 / 上期結存資料
出入庫紀錄表 (.xlsx)本期入庫 / 出庫資料；相容原本固定欄位順序
重量資料 (.xlsx)產品名稱 + 淨重 / 單位重量

② 月結總帳用檔案

這一區對應原本月結頁。月結不是重新讀原始租金表，而是讀「上半期租金試算結果」與「下半期租金試算結果」，再搭配既有總帳。如果你剛剛已用本頁分別算完上期、下期，可以不用再上傳，系統會先讀暫存的兩期試算。

上半期租金計算結果 (.xlsx)原本 ledger_file_first
下半期租金計算結果 (.xlsx)原本 ledger_file_second
本期總檔 (.xlsx，危險操作)原本用來覆蓋 Supabase 總帳；非一般月結必填
新版後續應改成：讀取上/下期 → 月結預覽 → 人工確認 → 更新總帳；不要直接覆蓋正式總帳。

③ 利潤分析用檔案

這一區對應原本利潤分析頁。必填是利潤明細表；租金計算結果與加工單是選配，但會影響租金單價來源。

利潤明細表 (.xlsx)原本 profit_file；欄位預設：產品、產品名稱、批號、數量
租金計算結果 (.xlsx，選配)原本 rental_result_file；用來帶出銷貨分攤單價
加工單 (.xlsx，選配)原本 processing_form_file；用來計算加工成品租金單價

④ 轉加工 / 在途歷史選配

轉加工頁另有加工單入口；在途銷貨台帳另有歷史 Excel / CSV 匯入入口。這些不是租金計算必填檔。

轉加工加工單 (.xlsx)原本 pg_order_file
在途歷史 Excel / CSV原本 transitHistoryFileInput
