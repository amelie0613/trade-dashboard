# 統一協調層 (RentalAnalysisPipeline) 使用指南

## 📋 概述

`dodo6123.html` 是改進版本，集成了 **RentalAnalysisPipeline (協調層)**，將租金分析流程統一為一個流水線操作，解決了原始 `dodo6122.html` 中分散執行、數據不一致的問題。

## ✨ 新增特性

### 1. **統一的流水線執行**
將以下 5 個階段串聯執行：
```
Stage 1: 租金計算 → Stage 2: 出庫分類 → Stage 3: 加工分析 
  → Stage 4: 利潤匯總 → Stage 5: 在途核銷
```

### 2. **一鍵完成分析**
點擊「🚀 完整分析」按鈕，即可執行整個流程，無需手動操作各個頁面。

### 3. **實時進度反饋**
- 顯示當前執行的階段
- 實時進度條
- 完整的執行日誌

### 4. **智能錯誤處理**
- 可選的 Stage 失敗時自動跳過（如加工單）
- 必需的 Stage 失敗時自動中止
- 詳細的錯誤報告

### 5. **數據驗證檢查點**
每個 Stage 完成後都會進行驗證：
- 必要欄位完整性
- 計算結果合理性
- 比對率檢查

## 🚀 如何使用

### 快速開始

1. **打開 dodo6123.html**
   ```
   在瀏覽器中打開 dodo6123.html
   ```

2. **上傳必需的檔案**
   - 租金資料表 (.xlsx)
   - 出入庫記錄表 (.xlsx)
   - 重量資料表 (.xlsx) - 如果有單位重量欄位

3. **設定計算參數**
   - 期初日期：計算開始日期
   - 期末日期：計算結束日期
   - 日租金：每 15 天/kg 的費用
   - 出入庫費用：每次操作/kg 的費用

4. **（可選）上傳加工單**
   - 在利潤分析頁面上傳加工單
   - 系統自動計算成品單價

5. **（可選）上傳利潤明細表**
   - 在利潤分析頁面上傳利潤明細表
   - 系統自動進行租金試算

6. **點擊「🚀 完整分析」**
   系統將自動執行所有 5 個階段，並在最後顯示結果摘要。

## 📊 執行流程詳解

### Stage 1: 租金計算
```
輸入：租金資料表、出入庫記錄、重量資料、日期範圍、費率
輸出：
  - 各產品的租金成本（前期結餘 + 本期入庫）
  - 出入庫費用
  - 出庫詳細記錄
驗證：
  ✓ 必要欄位完整性
  ✓ 費用合理性
  ✓ 無負值檢查
```

### Stage 2: 出庫分類
```
輸入：Stage 1 的出庫清單
輸出：
  - 按出庫類型分類（銷售、調撥、拆解）
  - 出庫詳細統計
驗證：
  ✓ 出庫數量有效性
  ✓ 分類完整性
```

### Stage 3: 加工單分析 (可選)
```
輸入：加工單、Stage 1 的租金結果
輸出：
  - 各成品批號的單位租金成本
  - 母件→成品的成本分攤計算
驗證：
  ✓ 母件找到單價的比例
  ✓ 子件數量一致性
```

### Stage 4: 利潤匯總
```
輸入：利潤明細表、前 3 個 Stage 的結果
輸出：
  - 帶有租金單價的利潤明細表
  - 比對情況統計
優先級：加工成品 > 租金結果 > 總帳分攤單價 > 在途台帳
驗證：
  ✓ 必要欄位配對
  ✓ 租金來源有效性
  ✓ 比對率檢查
```

### Stage 5: 在途核銷 (可選)
```
輸入：Stage 4 的利潤結果、在途銷貨台帳
輸出：
  - 核銷記錄更新
  - 在途銷貨台帳更新（Supabase）
驗證：
  ✓ FIFO 邏輯正確性
  ✓ 餘額計算正確性
```

## 🔍 錯誤診斷

### 常見錯誤及解決方案

| 錯誤 | 原因 | 解決方案 |
|------|------|--------|
| `租金資料不能為空` | 未上傳租金表 | 在「上傳檔案」區域上傳租金資料表 |
| `出入庫記錄不能為空` | 未上傳出入庫表 | 上傳出入庫記錄表 |
| `期初日期必須小於期末日期` | 日期設定錯誤 | 確保期初 < 期末 |
| `請先在利潤分析頁面上傳並試算利潤明細表` | 未執行利潤試算 | 先在「利潤分析」頁面上傳並點擊「試算」 |
| `加工單未上傳（可選，已跳過）` | 加工單可選 | 如需使用，上傳加工單；否則可忽略此警告 |

## 💡 提示

### 數據流向檢查清單

- [ ] **必須先執行租金計算** - 確保 Stage 1 成功完成
- [ ] **如要使用加工單** - 上傳加工單後才執行完整分析
- [ ] **如要試算利潤** - 先在利潤分析頁面上傳並手動試算
- [ ] **核銷在途銷貨** - 需要有 Supabase 連線和有效的在途台帳

### 執行順序建議

#### 基本流程 (無加工單)
```
1. 打開 dodo6123.html
2. 上傳租金表和出入庫表
3. 點擊「完整分析」
4. 查看結果
```

#### 完整流程 (含加工單和利潤分析)
```
1. 打開 dodo6123.html
2. 上傳租金表和出入庫表
3. 切換到「利潤分析」頁面
4. 上傳利潤明細表和加工單
5. 點擊「試算租金費用」(手動驗證)
6. 回到「租金計算」頁面
7. 點擊「完整分析」 ← 執行完整流程
```

## 📝 技術細節

### 協調層架構

```javascript
class RentalAnalysisPipeline {
  // 配置輸入
  constructor(config)
  
  // 驗證和執行
  async run() {
    // 1. 驗證輸入
    // 2. 依次執行 5 個 Stage
    // 3. 收集結果和驗證信息
    // 4. 返回匯總報告
  }
  
  // 註冊回調
  onProgress(callback)    // 進度更新
  onError(callback)       // 錯誤處理
}
```

### Stage 結果格式

每個 Stage 返回統一的結果格式：

```javascript
{
  stage: 'stage1_rental',           // 階段識別
  data: [ ... ],                    // 實際結果數據
  summary: {
    totalRows: 150,                 // 總筆數
    processedRows: 148,             // 已處理筆數
    key_metrics: { ... }            // 關鍵指標
  },
  validation: {
    status: 'PASS|WARNING|ERROR',
    checks: [ ... ]                 // 驗證檢查結果
  }
}
```

## 🔄 與 dodo6122.html 的差異

| 功能 | dodo6122.html | dodo6123.html |
|------|---------------|---------------|
| 執行方式 | 手動分頁執行 | 一鍵完整流程 |
| 數據流 | 分散、依賴全局變量 | 統一管道、清晰追蹤 |
| 進度反饋 | 無 | 實時進度條和日誌 |
| 錯誤恢復 | 無 | 智能跳過可選 Stage |
| 驗證機制 | 無 | 每 Stage 都有檢查點 |
| 源文件保護 | N/A | 獨立新檔案 |

## 🎯 最佳實踐

1. **定期備份數據** - 執行前備份 Excel 檔案
2. **檢查日期範圍** - 確保期初/期末日期設定正確
3. **驗證輸入數據** - 上傳前確認 Excel 格式正確
4. **查看完整報告** - 執行後檢查警告和錯誤詳情
5. **保留執行日誌** - 如有問題，保存控制台輸出便於診斷

## 📞 支援

如遇到問題：

1. **查看控制台日誌**
   - 按 F12 打開開發者工具
   - 查看 Console 標籤的錯誤信息

2. **檢查驗證報告**
   - 執行完成後查看 UI 顯示的警告和錯誤列表
   - 根據失敗的 Stage 排查問題

3. **驗證輸入數據**
   - 確認 Excel 檔案格式正確
   - 檢查必需的欄位是否存在
   - 驗證日期格式（YYYYMMDD 或 YYYY-MM-DD）
